# BSD Partition

FreeBSD 在 MBR 里常见的类型值是 `0xA5`，通常先是一个 **FreeBSD slice**，然后在 slice 内再放一个 **BSD disklabel**，disklabel 里继续划分 `a/b/c/d/e...` 这些 BSD 子分区。

## BSD slice

从 MBR 的角度看，它和普通 MBR 分区没有区别，区别主要在 `partition_type=0xA5`，表示这一整个分区内部还要继续按 BSD 的方式解释。

## BSD 子分区条目

disklabel 的分区表从 `0x94` 开始，每个条目 16 字节，常见命名是 `a`、`b`、`c`、`d`、`e`。

| 偏移 | 大小 | 含义         | 默认初始值                                                   |
| ---- | ---- | ------------ | ------------------------------------------------------------ |
| 0    | 4    | p_size       | 子分区大小，单位通常是扇区                                   |
| 4    | 4    | p_offset     | 子分区相对 BSD slice 起点的 LBA 偏移                         |
| 8    | 4    | p_fsize      | 文件系统基本块/片段大小                                      |
| C    | 1    | p_fstype     | 文件系统类型，常见 `1=swap`, `7=ufs`, `27=zfs`               |
| D    | 1    | p_frag       | 每块包含的 fragment 数量                                     |
| E    | 2    | p_cpg        | 每 cylinder group 的柱面数                                   |

常见约定：

- `a` 常作为根文件系统，例如 `/`
- `b` 常作为 `swap`
- `c` 常表示整个 slice 的覆盖分区，不一定拿来挂载
- `d/e/...` 常作为额外的文件系统，比如 `/var`

# UFS / UFS2

UFS 是 BSD 系统常见的本地文件系统，FreeBSD 里现代镜像更常见的是 **UFS2**。它和 ext 一样有超级块、inode、目录项这些基础概念，但字段布局和块寻址方式不同。

## 超级块位置

UFS/UFS2 不像 ext 固定在分区起始后 `1024` 字节开始。常见的超级块候选位置是：

- `0x2000`
- `0x10000`

UFS2 常见魔数位于超级块内偏移 `0x55C`：

- `UFS1_MAGIC = 0x00011954`
- `UFS2_MAGIC = 0x19540119`

下面列的是分析、遍历目录、定位 inode 时最常用的一部分字段。

| 10   | 4    | fs_iblkno           | cylinder group 内 inode 区起始 fragment 编号                |
| 30   | 4    | fs_bsize            | 文件系统块大小，例如 16384                                   |
| 34   | 4    | fs_fsize            | fragment 大小，例如 2048                                     |
| 38   | 4    | fs_frag             | 每个 block 含多少个 fragment                                 |
| 74   | 4    | fs_nindir           | 一个间接块中可容纳多少个块指针                               |
| 78   | 4    | fs_inopb            | 每个 block 中可容纳多少个 inode                              |
| B8   | 4    | fs_ipg              | 每个 cylinder group 中的 inode 数量                          |

## cylinder group 与 inode 定位

UFS2 的 inode 不是简单线性排布的，而是按 cylinder group 分组。

常用的定位思路：

1. `cg = inode_num // fs_ipg`
2. `local = inode_num % fs_ipg`
3. `fragno = cg * fs_fpg + fs_iblkno + (local // fs_inopb) * fs_frag`
4. `inode_offset = start_base + fragno * fs_fsize + (local % fs_inopb) * inode_size`

在 UFS2 里常见：

- `inode_size = 256`
- 根目录 inode 通常是 `2`

## UFS2 目录项

UFS2 目录是一个变长记录数组，每条记录常见结构如下：

| 偏移 | 大小 | 含义     | 默认初始值                                                   |
| ---- | ---- | -------- | ------------------------------------------------------------ |
| 0    | 4    | d_fileno | 指向的 inode 编号                                            |
| 4    | 2    | d_reclen | 当前目录项总长度                                             |
| 6    | 1    | d_type   | 文件类型，例如 `4=dir`, `8=file`, `10=link`                  |
| 7    | 1    | d_namlen | 文件名长度                                                   |
| 8    | *    | d_name   | 文件名数据                                                   |

目录遍历时通常流程是：

1. 找到目录 inode
2. 读取 inode 对应的数据块
3. 从块内按 `d_reclen` 逐条推进
4. 取出 `d_fileno`、`d_type`、`d_name`

## UFS2 块寻址

UFS2 的数据块读取逻辑和 ext 类似，也是：

- 先读 `di_db[12]` 的直接块
- 再读 `di_ib[0]` 的单间接
- 再读 `di_ib[1]` 的双间接
- 再读 `di_ib[2]` 的三间接

区别在于 UFS2 的块号通常是 **64 位**，而且块地址换算时经常需要乘的是 `fs_fsize`（fragment 大小），不是简单的 512。

## UFS2 常见文件标志

`di_flags` 里常见的值：

- `UF_NODUMP = 0x00000001`
- `UF_IMMUTABLE = 0x00000002`
- `UF_APPEND = 0x00000004`
- `UF_OPAQUE = 0x00000008`
- `SF_ARCHIVED = 0x00010000`
- `SF_IMMUTABLE = 0x00020000`
- `SF_APPEND = 0x00040000`
- `SF_NOUNLINK = 0x00100000`
- `SF_SNAPSHOT = 0x00200000`

# EXT

**ext2**  不支持日志（journal）功能 → `s_feature_incompat`、`s_feature_ro_compat`、`s_feature_compat` 没有日志标志

**ext3**  支持日志（journal） → 超级块字段 `s_feature_incompat` 或 `s_feature_compat` 中有 `has_journal` 标志

**ext4**  支持更高级特性 → `s_feature_incompat` 有 `extents`、`64bit` 等标志，支持大文件和更大的分区

## 区块组描述符表

块组描述符表是一个 块组描述符数组，用于定义所有 块组的参数。它提供了 inode 位图和 inode 表的位置、块位图、空闲块和 inode 的数量以及其他一些有用的信息。 块组描述符表从超级块之后的第一个块开始。在 1KiB 块文件系统中，这相当于第三个块；而在 2KiB 及更大的块文件系统中，则相当于第二个块。块组描述符表的卷影副本也与超级块的每个副本一起存储。

## 索引节点表

inode 表用于跟踪每个目录、普通文件、符号链接或特殊文件；它们的位置、大小、类型和访问权限都存储在 inode 中。inode 本身不存储文件名，文件名仅包含在 目录文件中。每个块组都有一个 inode 表，可以通过读取 其关联的组描述符中的bg_inode_table来定位。每个表有s_inodes_per_group 个 inode。每个 inode 都包含系统中单个物理文件的信息。文件可以是目录、套接字、缓冲区、字符设备或块设备、符号链接或普通文件。因此，inode 可以被视为与某个实体相关的信息块，描述了该实体在磁盘上的位置、大小和所有者。
